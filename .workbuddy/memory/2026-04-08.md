```python
# 准备指令

    Instruction(id=0, opcode="add", defs=[0], uses=[], live_out={0}),
]

# 执行分配
result = allocator.allocate(instructions)
```

## 寄存器分配器架构重构完成

### 背景
用户提出架构问题：寄存器分配器放在 `codegen` 模块，后续废弃 `codegen` 时会发生什么？

### 问题分析
1. **模块定位错误**：寄存器分配是 IR 层优化，不应该属于代码生成层
2. **依赖关系混乱**：`codegen` 模块依赖 `zhc.parser`，导致循环导入风险
3. **迁移成本高**：废弃 `codegen` 时需要修改所有导入路径

### 重构方案
将寄存器分配器从 `codegen` 移动到正确的架构位置：

```
src/
├── ir/                       # IR 层优化
│   ├── ssa.py               # SSA 构建
│   ├── dataflow.py          # 数据流分析
│   ├── loop_optimizer.py    # 循环优化
│   ├── inline_optimizer.py  # 内联优化
│   └── register_allocator.py # 寄存器分配 ← 移到这里
│
├── backend/                  # 后端层
│   ├── __init__.py          # 后端模块初始化
│   └── allocator_interface.py # 后端适配接口
│
└── codegen/                  # 临时模块（已废弃）
    ├── __init__.py          # 废弃警告 + 重新导出
    ├── register_allocator.py # 废弃包装器
    └── allocator_interface.py # 废弃包装器
```

### 实施步骤
1. ✅ 创建 `src/backend/` 目录结构
2. ✅ 移动 `register_allocator.py` 到 `src/ir/`
3. ✅ 移动 `allocator_interface.py` 到 `src/backend/`
4. ✅ 更新所有导入路径
5. ✅ 更新测试文件
6. ✅ 添加废弃警告到 `codegen` 模块
7. ✅ 运行测试验证重构
8. ✅ 更新文档

### 架构优势
1. **清晰的分层**：IR 层优化 vs 后端代码生成
2. **避免循环依赖**：IR 层不依赖 parser
3. **易于迁移**：废弃 codegen 时不受影响
4. **符合编译器架构**：SSA、数据流、寄存器分配都是 IR 层优化
