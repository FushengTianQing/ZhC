## 三、自动补全配置 (T030)

### 3.1 文件信息

- **文件**: `ide/vscode/completions.json`
- **大小**: 2.1 KB
- **格式**: JSON

### 3.2 补全分类

将补全项分为 **8个类别**，便于组织和管理：

#### 1) 关键字 (keywords)
- 模块、导入、公开、私有、保护
- 共 **5项**

#### 2) 函数 (functions)
- 函数、主函数、返回、构造函数、析构函数
- 共 **5项**

#### 3) 类型 (types)
- 整数型、浮点型、双精度型、字符型、字符串型、布尔型、空型、长整型
- 共 **8项**

#### 4) 控制流 (controlFlow)
- 如果、如果否则、否则、循环、当、跳出、继续、开关、情况、默认
- 共 **10项**

#### 5) 类相关 (classes)
- 类、继承、虚函数、静态
- 共 **4项**

#### 6) 常量 (constants)
- 真、假、空
- 共 **3项**

#### 7) 标准库 (stdlib)
- 打印、长度、连接、子串、查找、替换
- 共 **6项**

#### 8) 变量声明 (variables)
- 整数变量、浮点变量、字符串变量、数组、指针、常量
- 共 **6项**

### 3.3 补全项结构

每个补全项包含：

```json
{
  "label": "显示文本",
  "kind": "类型（Keyword/Function/Class等）",
  "insertText": "插入文本（支持占位符）",
  "documentation": "文档说明",
  "sortText": "排序优先级"
}
```

### 3.4 特色功能

1. **智能占位符**: 使用 `${1:默认值}` 语法
2. **排序优先级**: 按类别排序（a-h）
3. **完整文档**: 每项都有详细说明
4. **标准库集成**: 包含string和stdio库函数

## 四、错误诊断规则 (T031)

### 4.1 文件信息

- **文件**: `ide/vscode/diagnostics.md`
- **大小**: 8.5 KB
- **格式**: Markdown

### 4.2 错误级别

- **Error**: 15种，必须修复
- **Warning**: 4种，建议修复
- **Hint**: 1种，改进建议

### 4.3 错误分类

#### 一、语法错误 (ZHCE0001-ZHCE0004)
1. `ZHCE0001` - 括号不匹配
2. `ZHCE0002` - 分号缺失
3. `ZHCE0003` - 字符串未闭合
4. `ZHCE0004` - 注释未闭合

#### 二、类型错误 (ZHCE0101-ZHCE0103)
5. `ZHCE0101` - 类型不匹配
6. `ZHCE0102` - 返回类型不匹配
7. `ZHCE0103` - 参数类型错误

#### 三、声明错误 (ZHCE0201-ZHCE0203)
8. `ZHCE0201` - 重复声明
9. `ZHCE0202` - 未声明使用
10. `ZHCE0203` - 作用域错误

#### 四、模块错误 (ZHCE0301-ZHCE0303)
11. `ZHCE0301` - 模块未找到
12. `ZHCE0302` - 循环依赖
13. `ZHCE0303` - 访问权限错误

#### 五、类相关错误 (ZHCE0401-ZHCE0402)
14. `ZHCE0401` - 继承错误
15. `ZHCE0402` - 构造函数错误

#### 六、警告 (ZHCW0601-ZHCW0604)
16. `ZHCW0601` - 未使用变量
17. `ZHCW0602` - 代码不可达
18. `ZHCW0603` - 空代码块
19. `ZHCW0604` - 魔法数字

### 4.4 文档结构

每个错误包含：

- **规则ID**: 唯一标识
- **级别**: Error/Warning/Hint
- **描述**: 简短说明
- **示例**: 错误代码示例
- **修复建议**: 如何修复

### 4.5 实现特性

1. **实时诊断**: 编辑、保存、切换文件时触发
2. **增量分析**: 只检查修改的部分
3. **异步执行**: 不阻塞UI
4. **快速修复**: 部分错误提供自动修复
