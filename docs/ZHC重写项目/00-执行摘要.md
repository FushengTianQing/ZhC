## 三件事（核心交付）

### 1. C++/LLVM 重写（Phase 1-2）
把 Python 前端全部重写为 C++，直接接入 LLVM 生态，不再依赖外部编译器。

### 2. 可视化执行追踪（Phase 0+3，MVP 先行）
```
zhc run --trace 程序.zhc
→ trace.json（结构化执行轨迹）
→ trace.html（中文可视化页面）
```
让学生和老师看到 AI 生成的代码"实际做了什么"，而不是"看起来做什么"。

### 3. AI 编程接口 + 可信监控（Phase 4-5）
编译器内嵌 AI 助手（不是插件），加上一层可信执行监控——**在 AI 代码执行之前先过安检**。

## 战略赌注

| 赌注 | 理由 |
|:---|:---|
| **教育优先于专业** | AI 写代码的断代人最痛，付费意愿最强，验证周期最短 |
| **中文是护城河** | 全球唯一中文 C 编译器，竞品无法复制 |
| **可视化追踪是入口** | 先用 trace.html 把用户拉进来，AI 功能后续叠加 |

## FAQ（决策者版）

**Q: 为什么不用 Rust 重写？**
A: Rust 有自己的编译器（rustc），我们没有语言层面的差异化；C++ 是 LLVM 生态的标准语言，Clang 就是 C++ 写的。

**Q: 14 个月后如果 Copilot 也做了可视化追踪怎么办？**
A: 那是好事，说明市场被教育了。ZhC 的优势是：中文 + 编译器内嵌（不是插件）+ 教育场景深耕。

**Q: 为什么是 2 人而不是 3 人？**
A: 3 人理论加速 40%，但沟通成本增加 60%，实际产出未必更多。2 人全职，一人主前端，一人主 AI/后端，边界清晰。
