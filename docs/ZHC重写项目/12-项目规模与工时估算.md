## 12.5 各阶段 Go/No-Go 检查点

> 每个阶段结束时必须通过 Go/No-Go 检查点，未通过则进入问题修复阶段，不得自动进入下一阶段。

| 阶段 | Go 条件 | No-Go 条件 | 修复策略 |
|:---|:---|:---|:---|
| **Phase 0 → Phase 1** | trace.html 可用，2 个程序跑通 | trace 生成失败 | 增加 Phase 0 工时（最多 +50%） |
| **Phase 1 → Phase 2** | C++ Lexer/Parser/AST 跑通现有测试集 | 解析错误率 > 5% | 停下手头开发，集中修 Parser Bug |
| **Phase 2 → Phase 3** | 能编译运行 fibonacci.zhc（递归版本） | 编译崩溃或输出错误 | 停 1 周专门修 CodeGen Bug |
| **Phase 3 → Phase 4** | trace.html 含变量追踪 + 调用栈 | HTML 渲染错误 | 增加 HTML 生成工时 |
| **Phase 4 → Phase 5** | AI 错误解释响应时间 < 5s | API 超时率 > 10% | 增加超时处理 + 本地模型降级 |
| **Phase 5 → Phase 6** | 沙箱执行基础可用 | 沙箱崩溃率 > 5% | 增加沙箱隔离测试 |
| **Phase 6 → v1.0** | .zhc/.c/.py 均能编译运行 | 任何一种语言失败 | 按优先级推迟 Python 前端 |
