## 1. converter/code.py 重构

### 问题
- 603行代码
- 错误处理与转换逻辑耦合
- add_error/add_warning方法内嵌在CodeConverter中

### 解决方案
创建独立的错误处理器 `conversion_error_handler.py`:

```python
class ConversionErrorHandler:
    """转换错误处理器"""
    def add_error(self, error_type, message, line_no): ...
    def add_warning(self, warning_type, message, line_no): ...
    def get_errors(self): ...
    def generate_report(self) -> str: ...
```

### 重构结果
- CodeConverter专注于转换逻辑
- 错误处理委托给ConversionErrorHandler
- 保持向后兼容（errors/warnings属性）

## 3. parser/parser.py 重构

### 问题
- 1047行代码
- 58个方法，职责过多
- 声明/语句/表达式解析混在一起

### 解决方案
拆分为3个子模块：

| 文件 | 行数 | 职责 |
|-----|-----|-----|
| declarations.py | ~250 | 模块、导入、函数、结构体、变量、常量、参数声明 |
| statements.py | ~200 | 块、if、while、for、break、continue、return、表达式语句 |
| expressions.py | ~300 | 类型、表达式、赋值、逻辑、比较、算术、一元、后缀、基本表达式 |

### 重构结果
```python
# 使用混入类组合
class DeclarationParserMixin:
    """声明解析混入类"""

class StatementParserMixin:
    """语句解析混入类"""

class ExpressionParserMixin:
    """表达式解析混入类"""
```

Parser主类保持精简，可通过多重继承组合功能。
