# 错误处理规范
## 中文C编译器模块系统错误处理标准

### 📋 概述

本文档定义了中文C编译器模块系统的错误处理标准，包括错误分类、错误报告、错误恢复和错误处理策略。确保系统在遇到各种异常情况时能够优雅地处理并提供有用的反馈。

### 🎯 设计目标

1. **用户友好**: 错误信息清晰易懂，提供修复建议
2. **开发友好**: 错误信息包含足够调试信息
3. **系统健壮**: 错误不应导致系统崩溃
4. **可恢复性**: 尽可能从错误中恢复并继续处理
5. **一致性**: 统一的错误处理模式和格式

### 📊 错误分类

#### 1. 按严重程度分类

| 等级 | 代码 | 描述 | 处理策略 |
|:---|:---|:---|:---|
| **信息** | INFO | 信息性消息，不影响处理 | 记录日志，可选显示 |
| **警告** | WARNING | 潜在问题，不影响继续处理 | 显示警告，继续处理 |
| **错误** | ERROR | 实际错误，可能影响结果质量 | 显示错误，尝试恢复 |
| **致命错误** | FATAL | 严重错误，无法继续处理 | 立即停止，显示错误 |

#### 2. 按错误类型分类

##### 语法错误 (SYNTAX)
| 错误代码 | 描述 | 示例 | 修复建议 |
|:---|:---|:---|:---|
| SYNTAX_MISSING_BRACE | 缺少花括号 | `模块 测试 {` | 添加对应的 `}` |
| SYNTAX_UNEXPECTED_TOKEN | 意外的标记 | `模块 { 测试 }` | 检查语法格式 |
| SYNTAX_INVALID_MODULE_DECL | 无效的模块声明 | `模块123 {` | 使用 `模块 模块名 {` |
| SYNTAX_INVALID_IMPORT_STMT | 无效的导入语句 | `导入 ;` | 使用 `导入 模块名;` |
| SYNTAX_INVALID_VISIBILITY | 无效的可见性修饰符 | `公开 {` | 使用 `公开:` |

##### 语义错误 (SEMANTIC)
| 错误代码 | 描述 | 示例 | 修复建议 |
|:---|:---|:---|:---|
| SEMANTIC_DUPLICATE_SYMBOL | 符号重复定义 | `整数型 x; 整数型 x;` | 使用不同的符号名 |
| SEMANTIC_UNDEFINED_SYMBOL | 未定义的符号 | `使用 未定义变量;` | 定义符号或导入模块 |
| SEMANTIC_TYPE_MISMATCH | 类型不匹配 | `整数型 x = "字符串";` | 使用正确的类型 |
| SEMANTIC_INVALID_RETURN | 无效的返回语句 | 在void函数中返回值 | 移除返回值或修改返回类型 |

##### 作用域错误 (SCOPE)
| 错误代码 | 描述 | 示例 | 修复建议 |
|:---|:---|:---|:---|
| SCOPE_VIOLATION | 作用域违规 | 在模块外访问私有符号 | 将符号设为公开或导入模块 |
| SCOPE_OUT_OF_SCOPE | 超出作用域访问 | 在函数外访问局部变量 | 将变量移到合适的作用域 |
| SCOPE_INVALID_ACCESS | 无效的访问控制 | 访问其他模块的私有符号 | 使用公开接口或修改可见性 |

##### 依赖错误 (DEPENDENCY)
| 错误代码 | 描述 | 示例 | 修复建议 |
|:---|:---|:---|:---|
| DEPENDENCY_CYCLE | 循环依赖 | A导入B，B导入A | 重构模块消除循环依赖 |
| DEPENDENCY_MISSING_MODULE | 缺失的模块依赖 | `导入 不存在的模块;` | 创建模块或检查模块名 |
| DEPENDENCY_VERSION_CONFLICT | 版本冲突 | 模块版本不兼容 | 使用兼容的模块版本 |

##### 编译错误 (COMPILE)
| 错误代码 | 描述 | 示例 | 修复建议 |
|:---|:---|:---|:---|
| COMPILE_CONVERSION_FAILED | 转换失败 | 无法转换复杂表达式 | 简化表达式或手动转换 |
| COMPILE_UNSUPPORTED_FEATURE | 不支持的功能 | 使用了未实现的功能 | 使用替代方案或等待支持 |

##### 系统错误 (SYSTEM)
| 错误代码 | 描述 | 示例 | 修复建议 |
|:---|:---|:---|:---|
| SYSTEM_FILE_NOT_FOUND | 文件未找到 | 无法打开输入文件 | 检查文件路径和权限 |
| SYSTEM_IO_ERROR | I/O错误 | 读写文件失败 | 检查磁盘空间和权限 |
| SYSTEM_MEMORY_ERROR | 内存错误 | 内存分配失败 | 减少文件大小或增加内存 |

### 📝 错误记录格式

#### 标准错误记录
```python
{
    "type": "SYNTAX_INVALID_MODULE_DECL",  # 错误类型代码
    "message": "无效的模块声明",           # 错误描述
    "line": 10,                           # 行号（-1表示未知）
    "column": 5,                          # 列号（-1表示未知）
    "severity": "ERROR",                  # 严重程度
    "context": "模块 {",                  # 错误上下文
    "suggestion": "使用 '模块 模块名 {'",  # 修复建议
    "timestamp": "2026-04-03T20:00:00Z"   # 时间戳
}
```

#### 错误显示格式
```plaintext
[严重程度] 行号:列号: 错误类型 - 错误消息
   上下文: 错误代码行
   建议: 修复建议
```

**示例输出**:
```
[错误] 行10:5: 无效的模块声明 - 缺少模块名
   上下文: 模块 {
   建议: 使用 '模块 模块名 {'
```

### 🔧 错误处理策略

#### 1. 语法错误处理
- **检测时机**: 解析阶段立即检测
- **恢复策略**: 尝试跳过错误行继续解析
- **错误报告**: 提供具体位置和修复建议
- **最大错误数**: 达到阈值后停止处理

#### 2. 语义错误处理  
- **检测时机**: 解析完成后进行语义分析
- **恢复策略**: 标记错误但继续转换可能的部分
- **错误报告**: 提供详细的原因分析和建议
- **严重性分级**: 根据影响程度分类处理

#### 3. 系统错误处理
- **检测时机**: 操作执行时立即检测
- **恢复策略**: 根据错误类型决定是否重试
- **错误报告**: 提供系统级错误信息和解决方案
- **用户提示**: 指导用户解决系统问题

#### 4. 致命错误处理
- **触发条件**: 无法恢复的系统级错误
- **处理策略**: 立即停止所有处理
- **清理操作**: 释放资源，保存中间状态
- **错误报告**: 提供详细的错误堆栈和诊断信息

### 🛠️ 错误恢复机制

#### 1. 语法错误恢复
```python
def parse_with_recovery(lines, error_handler):
    """带错误恢复的解析函数"""
    for i, line in enumerate(lines, 1):
        try:
            parse_line(line, i)
        except SyntaxError as e:
            # 记录错误
            error_handler.add_error(
                ErrorType.SYNTAX_ERROR,
                str(e),
                i,
                context=line,
                suggestion=get_syntax_suggestion(e)
            )
            # 尝试恢复：跳过当前行或使用默认值
            if can_recover_from(e):
                continue  # 跳过错误行
            else:
                break  # 无法恢复，停止解析
```

#### 2. 语义错误恢复
- **符号重复定义**: 忽略后续定义，使用第一个定义
- **未定义符号**: 标记为待处理，可能在其他文件中定义
- **类型不匹配**: 尝试类型转换或使用默认值

#### 3. 系统错误恢复
- **文件未找到**: 尝试其他路径或提示用户
- **I/O错误**: 重试有限次数后放弃
- **内存错误**: 释放缓存，尝试简化处理
