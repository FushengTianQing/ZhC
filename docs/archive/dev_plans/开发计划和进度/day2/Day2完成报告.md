## 🏗️ 技术实现详情

### 模块解析器架构

```python
# 核心数据结构
class ModuleParser:
    - current_module: ModuleInfo  # 当前解析的模块
    - current_visibility: str     # 当前可见性（public/private/protected）
    - modules: dict              # 模块名 -> ModuleInfo
    - scope_stack: list          # 作用域栈
    
class ModuleInfo:
    - name: str                  # 模块名
    - public_symbols: set        # 公开符号
    - private_symbols: set       # 私有符号
    - protected_symbols: set     # 保护符号
    - imports: list              # 导入的模块
    - exports: list              # 导出的符号
```

### 作用域管理器架构

```python
# 四级作用域体系
ScopeType.GLOBAL    # 全局作用域（程序级别）
ScopeType.MODULE    # 模块作用域（模块级别）
ScopeType.FILE      # 文件作用域（文件级别）
ScopeType.BLOCK     # 块作用域（代码块级别）

# 可见性控制
Visibility.PUBLIC    # 对其他模块可见
Visibility.PRIVATE   # 仅对当前模块可见（默认）
Visibility.PROTECTED # 对当前模块和子模块可见
```

### 转换规则设计

1. **公开符号转换**: `模块名_符号名`
   - 输入: `数学库.加法`
   - 输出: `数学库_加法`

2. **私有符号转换**: `static 模块名_符号名`
   - 输入: `数学库.内部常量` (私有)
   - 输出: `static 数学库_内部常量`

3. **保护符号转换**: 暂时与私有相同（待扩展）
