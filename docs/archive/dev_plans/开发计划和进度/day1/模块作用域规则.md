## 作用域层次设计

### 1. 四级作用域体系

| 作用域层级 | 可见范围 | 关键词 | 示例 |
|:---|:---|:---|:---|
| **全局作用域** | 整个程序可见 | （无特殊修饰符） | `常量 GLOBAL = 10;` |
| **模块作用域** | 模块内可见 | `模块 模块名 { ... }` | `模块 工具 { 公开: 函数 f() { ... } }` |
| **文件作用域** | 单个.zhc文件内可见 | `静态` 或 `私有:`区域 | `静态 整数型 文件变量;` |
| **局部作用域** | 函数/代码块内可见 | `{ }` 内部定义 | `{ 整数型 局部变量 = 5; }` |

### 2. 访问控制规则

#### 公开（public:）
- **可见性**：模块内外均可访问
- **转换规则**：添加模块名前缀
- **示例**：
  ```c
  模块 数学 {
      公开: 函数 平方(参数 x) -> 整数型 { ... }
  }
  // 转换后：int 数学_平方(int x) { ... }
  ```

#### 私有（private:）
- **可见性**：仅模块内同一文件可见
- **转换规则**：添加`static`修饰符，不添加模块名前缀
- **示例**：
  ```c
  模块 数学 {
      私有: 静态 整数型 计数器 = 0;
  }
  // 转换后：static int 计数器 = 0;
  ```

#### 保护（protected:） - 可选特性
- **可见性**：模块内及嵌套模块可见
- **转换规则**：特殊前缀（如`_protected_模块名_`）
- **示例**：
  ```c
  模块 外层 {
      保护: 函数 受保护函数() { ... }
      
      模块 内层 {
          可使用 受保护函数();
      }
  }
  ```

## 作用域解析规则

### 1. 符号查找顺序
当引用一个符号时，按以下顺序查找：

```
局部作用域 → 文件作用域 → 模块作用域 → 导入模块作用域 → 全局作用域
```

### 2. 作用域遮蔽规则
- 内层作用域符号遮蔽外层同名符号
- 明确引用可使用`模块名::符号名`语法
- 导入的符号可能遮蔽本地符号（需警告）

### 3. 跨模块引用规则

#### 允许的跨模块引用
1. 模块A的`公开:`符号 → 可在模块B中通过`A::符号`访问
2. 模块A的`公开:`符号 → 可导入到模块B中直接使用
3. 嵌套模块的父模块符号 → 可在子模块中直接使用

#### 禁止的跨模块引用
1. 模块A的`私有:`符号 → 不能在模块B中访问
2. 模块A的`保护:`符号 → 不能在不相关的模块中访问
3. 文件作用域符号 → 不能在模块外访问（即使在同一模块）

## 可见性详细规则

### 1. 默认可见性
- 模块内定义的符号默认是什么可见性？
- **决策**：默认视为`私有:`，鼓励明确声明
- **理由**：增强封装性，减少意外暴露

### 2. 嵌套模块可见性继承

#### 规则设计
```
模块 外层 {
    公开:  符号1        // 内层可见
    私有:  符号2        // 内层不可见
    保护:  符号3        // 内层可见
    
    模块 内层 {
        可使用 符号1 和 符号3
        不可使用 符号2
    }
}
```

#### 转换实现
- 外层公开符号：内层自动添加外层前缀
- 外层保护符号：特殊前缀标记，内层可见
- 外层私有符号：完全隐藏

### 3. 导入符号可见性规则

#### 导入方式
```c
// 方式1：导入整个模块（所有公开符号）
导入 模块A;

// 方式2：导入特定符号
导入 模块A::符号1;

// 方式3：导入并重命名（可选）
导入 模块A 为 别名;
```

#### 可见性影响
1. **导入整个模块**：该模块所有公开符号在当前作用域可见
2. **导入特定符号**：仅指定符号可见
3. **导入重命名**：原模块名不可见，仅别名可用

## 命名冲突处理策略

### 1. 冲突检测规则

#### 必检测冲突
1. 同一作用域内同名符号
2. 导入符号与本地符号同名
3. 不同导入模块的同名公开符号

#### 可选检测冲突  
1. 不同层级作用域的遮蔽冲突
2. 转换后可能产生的C语言命名冲突

### 2. 冲突解决机制

#### 自动解决策略
1. **添加唯一标识符**：冲突符号添加随机后缀
2. **作用域限定**：强制使用完整路径`模块::符号`
3. **导入选择**：指定使用哪个导入符号

#### 开发者干预机制
1. **明确声明**：使用`别名`语法重命名
2. **导入过滤**：只导入需要的符号
3. **作用域调整**：调整定义位置避免冲突

### 3. 错误处理流程

```
检测到冲突 → 尝试自动解决 → 成功则继续
              ↓
        自动解决失败 → 生成错误报告
              ↓
        提示开发者干预 → 修改代码解决
```

## 作用域转换规则详细设计

### 1. 符号重命名规则

| 符号类型 | 原始定义 | 转换后名称 | 说明 |
|:---|:---|:---|:---|
| 公开函数 | `模块A { 公开: 函数 f() }` | `A_f()` | 模块名前缀 + 下划线 |
| 公开变量 | `模块A { 公开: 整数型 x; }` | `A_x` | 同上 |
| 私有函数 | `模块A { 私有: 函数 g() }` | `static g()` | 添加static，不加前缀 |
| 私有变量 | `模块A { 私有: 整数型 y; }` | `static y` | 同上 |
| 嵌套模块 | `A::B::f()` | `A_B_f()` | 多级前缀用下划线连接 |

### 2. 作用域解析实现

#### 解析步骤
1. **词法分析**：识别`模块`、`公开:`、`私有:`等关键词
2. **作用域栈管理**：维护当前作用域层级
3. **符号表构建**：记录符号的可见性和定义位置
4. **引用解析**：将符号引用解析为转换后名称

#### 数据结构设计
```python
class Scope:
    def __init__(self, name, level, parent=None):
        self.name = name          # 作用域名
        self.level = level        # 层级（0=全局，1=模块，2=嵌套...）
        self.parent = parent      # 父作用域
        self.symbols = {}         # 符号表
        
class Symbol:
    def __init__(self, name, type, visibility, location):
        self.name = name          # 原始名称
        self.type = type          # 函数/变量/常量等
        self.visibility = visibility  # 公开/私有/保护
        self.location = location  # 定义位置
        self.transformed_name = None  # 转换后名称
```

## 验证用例设计

### 用例1：基础可见性验证
```c
模块 测试模块 {
    公开: 函数 公开函数() -> 整数型 { 返回 1; }
    私有: 函数 私有函数() -> 整数型 { 返回 2; }
    
    函数 测试() -> 空型 {
        整数型 a = 公开函数();  // ✓ 可访问
        整数型 b = 私有函数();  // ✓ 可访问（同一模块）
    }
}

// 其他模块尝试访问
模块 其他模块 {
    函数 测试() -> 空型 {
        整数型 a = 测试模块::公开函数();  // ✓ 可访问
        整数型 b = 测试模块::私有函数();  // ✗ 编译错误
    }
}
```

### 用例2：嵌套模块可见性
```c
模块 外层 {
    公开: 函数 外层公开() { ... }
    保护: 函数 外层保护() { ... }
    私有: 函数 外层私有() { ... }
    
    模块 内层 {
        函数 测试() -> 空型 {
            外层公开();  // ✓ 可访问
            外层保护();  // ✓ 可访问
            外层私有();  // ✗ 编译错误
        }
    }
}
```

### 用例3：导入冲突处理
```c
模块 模块A {
    公开: 函数 冲突函数() { 返回 1; }
}

模块 模块B {
    公开: 函数 冲突函数() { 返回 2; }
}

模块 主程序 {
    导入 模块A;
    导入 模块B;
    
    函数 测试() -> 空型 {
        // 检测到冲突，需要明确指定
        整数型 a = 模块A::冲突函数();  // ✓ 明确指定
        整数型 b = 冲突函数();         // ✗ 歧义错误
    }
}
```
