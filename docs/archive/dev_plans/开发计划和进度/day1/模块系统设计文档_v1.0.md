## 设计目标

### 1.1 总体目标
为中文C编译器增加现代化的模块化编程能力，提高代码的组织性、复用性和可维护性。

### 1.2 具体目标

- **实现可行性**：在预处理器框架内可实现
- **扩展性**：为未来功能扩展预留空间

## 核心概念

### 2.1 模块（Module）
模块是代码组织的基本单元，包含一组相关的函数、变量、类型定义等。

**关键特性：**
- 封装性：模块内部实现对外隐藏
- 独立性：模块可独立编译和测试
- 可复用性：模块可在不同项目中复用

### 2.2 命名空间（Namespace）
模块在逻辑上创建一个命名空间，避免符号命名冲突。

### 2.3 符号（Symbol）
模块内定义的各种实体，包括：
- 函数、变量、常量
- 类型定义（结构体、枚举等）
- 嵌套模块

### 2.4 可见性（Visibility）
控制符号在模块内外的访问权限：
- **公开（Public）**：模块内外均可访问
- **私有（Private）**：仅模块内可见
- **保护（Protected）**：模块内及嵌套模块可见

## 语法规范

### 3.1 模块定义

#### 基础语法
```c
模块 模块名 {
    // 模块内容
}
```

#### 示例
```c
模块 数学工具 {
    公开:
        常量 PI = 3.1415926;
        
        函数 计算面积(参数 浮点型 半径) -> 浮点型 {
            返回 PI * 半径 * 半径;
        }
    
    私有:
        静态 整数型 调用次数 = 0;
}
```

### 3.2 访问控制区域

#### 语法格式
```
公开:
    符号定义列表

私有:
    符号定义列表

保护:
    符号定义列表
```

#### 规则说明
1. 每个区域以`公开:`、`私有:`或`保护:`开头
2. 区域持续到下一个区域开始或模块结束
3. 可定义多个区域，顺序任意
4. 未在区域内的符号默认视为`私有`

### 3.3 嵌套模块

#### 语法
```c
模块 外层 {
    公开: 函数 外层函数() { ... }
    
    模块 内层 {
        公开: 函数 内层函数() { ... }
    }
}
```

#### 可见性规则
- 外层公开符号：内层可见
- 外层保护符号：内层可见
- 外层私有符号：内层不可见

## 作用域与可见性

### 4.1 作用域层次

| 层级 | 范围 | 管理方式 |
|:---|:---|:---|
| 全局作用域 | 整个程序 | 无修饰符 |
| 模块作用域 | 模块内 | `模块 名 { ... }` |
| 文件作用域 | 单个文件 | `静态` 或 `私有:` |
| 局部作用域 | 函数/块内 | `{ ... }` |

### 4.2 符号查找顺序
```
1. 局部作用域（当前函数/块）
2. 文件作用域（当前文件）
3. 模块作用域（当前模块）
4. 导入模块作用域
5. 全局作用域
```

### 4.3 命名冲突处理

#### 冲突类型
1. 同一作用域内同名符号
2. 导入符号与本地符号同名
3. 不同导入模块的同名符号

#### 解决策略
1. **明确限定**：使用`模块名::符号名`
2. **别名导入**：`导入 模块::符号 为 别名`
3. **自动重命名**：添加唯一后缀（备选方案）

## 导入导出机制

### 5.1 导入语法

#### 基础导入
```c
导入 模块名;                    // 导入整个模块
```

#### 选择性导入
```c
导入 模块名::符号;              // 导入特定符号
导入 模块名::{符号1, 符号2};     // 导入多个符号
```

#### 重命名导入
```c
导入 模块名 为 别名;            // 模块别名
导入 模块名::符号 为 新名;       // 符号别名
```

### 5.2 导出机制

#### 区域导出（主要方式）
```c
模块 工具 {
    公开:                     // 此区域内符号自动导出
        函数 工具函数() { ... }
        常量 工具常量 = 100;
}
```

#### 显式导出（可选）
```c
模块 工具 {
    函数 内部实现() { ... }
    
    导出 内部实现 为 公开接口;  // 显式导出并重命名
}
```

### 5.3 依赖管理

#### 依赖解析
- 编译时静态解析
- 构建依赖图
- 检测循环依赖

#### 编译顺序
- 按依赖拓扑顺序编译
- 支持并行编译独立模块

## 编译与链接

### 6.1 编译流程

#### 单文件编译
```
源文件(.zhc) → 预处理器 → C文件(.c) → 编译器 → 目标文件(.o)
```

#### 多模块编译
```
模块A.zhc → A.c → A.o
模块B.zhc → B.c → B.o  
主程序.zhc → 主程序.c → 主程序.o
链接器 → 可执行程序
```

### 6.2 符号转换规则

| 符号类型 | 原始定义 | 转换后 | 链接属性 |
|:---|:---|:---|:---|
| 公开函数 | `模块::函数` | `模块_函数` | 全局可见 |
| 公开变量 | `模块::变量` | `模块_变量` | 全局可见 |
| 私有函数 | `模块内函数` | `static 函数` | 文件内可见 |
| 私有变量 | `模块内变量` | `static 变量` | 文件内可见 |

### 6.3 性能优化

#### 编译缓存
- 缓存已编译模块
- 依赖变化时自动重新编译

#### 增量编译
- 只编译修改的模块
- 减少大型项目编译时间

## 错误处理

### 7.1 语法错误

#### 模块定义错误
```
错误：模块定义语法错误，缺少 '{'
文件：math.zhc，第3行
建议：检查模块定义格式
```

#### 访问控制错误
```
错误：访问控制修饰符必须在模块内使用
文件：tool.zhc，第10行
建议：将 '公开:' 移到模块内部
```

### 7.2 语义错误

#### 模块未找到
```
错误：找不到模块 '不存在的模块'
文件：main.zhc，第5行
建议：检查模块名或添加搜索路径
```

#### 符号未找到
```
错误：模块 '数学' 中没有符号 '不存在的函数'
文件：main.zhc，第8行
建议：检查符号名，或查看模块公开接口
```

#### 循环依赖
```
错误：检测到循环依赖：模块A → 模块B → 模块A
文件：module_a.zhc，第2行
建议：重构设计，打破循环依赖
```

### 7.3 链接错误

#### 未定义符号
```
链接错误：未定义的符号 '数学_平方'
引用位置：main.zhc，第8行
定义位置：应为 math.zhc，但未找到
建议：检查模块编译和链接顺序
```

#### 重复定义
```
链接错误：符号 '工具_函数' 重复定义
定义1：tool1.zhc，第5行
定义2：tool2.zhc，第3行
建议：重命名冲突符号，或使用不同模块
```

## 实现架构

### 8.1 总体架构

```
源代码(.zhc)
    ↓
词法分析器
    ↓
语法分析器
    ↓
作用域管理器
    ↓
符号表
    ↓
转换器
    ↓  
C代码(.c)
    ↓
编译器
    ↓
可执行程序
```

### 8.2 核心组件

#### 模块解析器（ModuleParser）
- 识别模块定义和导入语句
- 构建模块依赖图
- 验证语法正确性

#### 作用域管理器（ScopeManager）
- 管理作用域层级栈
- 处理符号可见性
- 解析符号引用

#### 符号表（SymbolTable）
- 记录所有符号信息
- 管理符号重命名
- 支持快速查找

#### 转换器（Transformer）
- 应用符号转换规则
- 生成目标C代码
- 添加必要的前向声明

### 8.3 数据结构设计

#### 模块信息
```python
class ModuleInfo:
    name: str                    # 模块名
    file_path: str               # 源文件路径
    exports: Dict[str, Symbol]   # 导出符号
    imports: List[Import]        # 导入语句
    dependencies: List[str]      # 依赖模块
    scope: Scope                 # 模块作用域
```

#### 符号信息
```python
class Symbol:
    name: str                    # 原始名称
    type: SymbolType             # 函数/变量/常量/类型
    visibility: Visibility       # 公开/私有/保护
    module: str                  # 所属模块
    location: Location           # 定义位置
    transformed_name: str        # 转换后名称
```

#### 作用域信息
```python
class Scope:
    name: str                    # 作用域名
    level: int                   # 层级（0=全局）
    parent: Optional[Scope]      # 父作用域
    symbols: Dict[str, Symbol]   # 符号表
    children: List[Scope]        # 子作用域
```

### B. 转换示例

#### 原始代码
```c
模块 数学 {
    公开: 函数 平方(x) { 返回 x*x; }
    私有: 静态 计数器 = 0;
}

模块 主程序 {
    导入 数学;
    
    函数 主函数() {
        打印("%d", 数学::平方(5));
    }
}
```

#### 转换后C代码
```c
/* 数学模块 */
int 数学_平方(int x) { return x*x; }
static int 计数器 = 0;

/* 主程序模块 */
int 主函数() {
    printf("%d", 数学_平方(5));
    return 0;
}
```

### C. 设计决策记录

2. **显式导出语法**：提供额外灵活性
3. **模块接口文件**：分离声明和实现
