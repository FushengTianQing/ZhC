## Week 3: 异常处理重构 (2026-04-08 ~ 2026-04-14)

### 🎯 目标
建立统一的异常处理机制，消除异常处理不一致问题

### 📝 任务清单

#### Task 3.1: 设计异常类层次结构
**优先级**: P0 | **预计时间**: 2小时

**具体内容**:
1. 创建 `src/errors/` 目录
2. 设计异常基类 `ZHCError`
3. 创建子类: `LexerError`, `ParseError`, `SemanticError`, `CodeGenerationError`
4. 实现 `SourceLocation` 错误位置追踪
5. 实现错误上下文信息（源码片段、建议修复）

**产出物**:
- `src/errors/__init__.py`
- `src/errors/base.py` (ZHCError基类)
- `src/errors/lexer_error.py`
- `src/errors/parser_error.py`
- `src/errors/semantic_error.py`
- `src/errors/codegen_error.py`

**验收标准**:
- 所有异常类继承自统一基类
- 支持错误位置追踪
- 支持错误上下文信息
- 有完整的类型注解和文档字符串
