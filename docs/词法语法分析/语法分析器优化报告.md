## 二、问题分析

### 2.1 核心问题

**函数声明解析错误**

**问题现象**：
```python
代码: 整数型 add(整数型 a, 整数型 b) { 返回 a + b; }
错误: 4个解析错误
```

**根本原因**：

解析器期望的语法格式：
```
函数 类型 函数名(参数) { 函数体 }
```

但ZHC实际的语法格式：
```
类型 函数名(参数) { 函数体 }
```

这是两种不同的函数声明语法，导致解析失败。

### 2.2 其他问题

1. **类型解析不完整**
   - 未支持自定义类型
   - 错误处理不够友好

2. **错误恢复粗糙**
   - 恢复点不够全面
   - 错误信息不够详细

3. **测试覆盖不足**
   - 缺少边界情况测试
   - 缺少复杂类型测试

## 三、解决方案

### 3.1 函数声明解析修复

#### 方案设计

采用**前瞻检测**策略：

1. 遇到类型关键字时，向前查看
2. 如果是 `类型 标识符(`，判定为函数声明
3. 如果是 `类型 标识符`，判定为变量声明

#### 实现代码

```python
def parse_declaration(self) -> Optional[ASTNode]:
    """解析声明"""
    # ...
    
    # 函数声明或变量声明
    if self.match(TokenType.INT, TokenType.FLOAT, ...):
        # 向前查看：如果是 类型 标识符(，则是函数声明
        if self.peek_token().type == TokenType.IDENTIFIER and \
           self.peek_token(2).type == TokenType.LPAREN:
            return self.parse_function_decl_with_type()
        else:
            return self.parse_variable_decl()
```

#### 新增方法

```python
def parse_function_decl_with_type(self) -> FunctionDeclNode:
    """解析函数声明（新语法：类型 函数名(参数)）"""
    # 解析返回类型
    return_type = self.parse_type()
    
    # 函数名
    name = self.current_token().value
    self.expect(TokenType.IDENTIFIER, "期望函数名")
    
    # 参数列表
    self.expect(TokenType.LPAREN, "期望 '('")
    params = []
    if not self.match(TokenType.RPAREN):
        params.append(self.parse_param_decl())
        while self.match(TokenType.COMMA):
            self.advance()
            params.append(self.parse_param_decl())
    self.expect(TokenType.RPAREN, "期望 ')'")
    
    # 函数体
    body = None
    if self.match(TokenType.LBRACE):
        body = self.parse_block()
    
    return FunctionDeclNode(name, return_type, params, body, ...)
```

### 3.3 错误恢复增强

#### 恢复点扩展

```python
def synchronize(self):
    """错误恢复：跳到下一个语句"""
    while not self.is_at_end():
        # 在分号处恢复
        if self.current_token().type == TokenType.SEMICOLON:
            self.advance()
            return
        
        # 在关键字处恢复（扩展了关键字列表）
        if self.current_token().type in (
            TokenType.FUNCTION, TokenType.STRUCT,
            TokenType.IF, TokenType.ELSE, TokenType.WHILE, TokenType.FOR,
            TokenType.RETURN, TokenType.BREAK, TokenType.CONTINUE,
            TokenType.INT, TokenType.FLOAT,
            TokenType.CHAR, TokenType.BOOL, TokenType.VOID, TokenType.STRING,
            TokenType.BYTE, TokenType.CONST,
            TokenType.MODULE, TokenType.IMPORT, TokenType.EXPORT, TokenType.PRIVATE
        ):
            return
        
        # 在大括号处恢复（代码块结束）
        if self.current_token().type == TokenType.RBRACE:
            return
        
        self.advance()
```

### 3.4 代码结构优化

#### 新增辅助方法

```python
def consume_if_match(self, token_type: TokenType) -> bool:
    """如果匹配则消耗Token"""
    if self.current_token().type == token_type:
        self.advance()
        return True
    return False
```

#### 改进expect方法

```python
def expect(self, token_type: TokenType, message: str) -> Token:
    """期望特定类型的Token"""
    if self.current_token().type == token_type:
        return self.advance()
    
    error = ParseError(f"{message}, 但找到 '{self.current_token().value}'", 
                      self.current_token())
    self.errors.append(error)
    
    # 错误恢复：尝试跳过当前Token
    self.advance()
    return self.current_token()
```

def test_function_with_params(self):
