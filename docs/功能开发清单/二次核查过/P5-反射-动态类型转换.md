# P5-反射-动态类型转换

> **优先级**: P5
> **功能模块**: 反射
> **功能名称**: 动态类型转换
> **创建日期**: 2026-04-10
> **状态**: ✅ 已完成 (2026-04-11)

## 1. 功能概述

实现安全的运行时动态类型转换，支持向下转型、交叉转型、类型验证，并在转换失败时提供详细的错误信息。

### 1.1 目标

- ✅ 支持安全的向下转型（downcast）
- ✅ 支持交叉转型（crosscast）
- ✅ 支持类型转换验证
- ✅ 提供转换失败的详细诊断
- ✅ 支持 `as`/`is` 语法

### 1.2 语法设计

```zhc
// 基本动态转换
动物 a = 新 狗();

// 安全转换 - 使用 as 语法
狗 d = a 转为 狗;
如果 (d != 空) {
    d.叫();
}

// 类型检查 + 转换
如果 (a 是类型 狗) {
    狗 d = a 转为 狗;
    d.叫();
}

// 转换失败处理
猫 c = a 转为 猫;  // 返回 空，因为 a 实际是狗
如果 (c == 空) {
    打印("转换失败：a 不是猫");
}
```

### 2.2 设计亮点

1. **增强的错误信息**：CastError 包含错误类型、转换路径、祖先类型、建议替代
2. **CastValidator**：支持验证转换有效性、批量验证、最佳转换匹配、类型建议
3. **`as`/`is` 语法**：完整的 Lexer → Parser → AST → 语义分析 → IR 管线
4. **优先级正确**：`as`/`is` 在 `==`/`!=` 之下、比较运算符之上
5. **复用已有基础设施**：基于 TypeRegistry 和 TypeChecker，无额外状态

### 2.3 核心类

- **CastErrorType** 枚举：6 种错误类型
- **CastError** 数据类：错误详情 + 序列化支持
- **TypeCast** 增强：get_cast_path()、can_cast()
- **CastValidator**：validate()、find_best_cast()、get_suggestions()
- **AsExprNode/IsExprNode**：AST 节点
