## 2. 泛型系统集成任务

### P0 — 必须完成（阻塞端到端编译）

#### 任务 G.01 — GenericResolver 核心逻辑实现

- **优先级**: 🔴 P0
- **文件**: `src/zhc/semantic/generics.py`
- **现状**: `GenericResolver` 类存在但所有方法为空
- **需实现的方法**:

  | 方法签名 | 当前状态 | 工作量 |
  |---------|---------|--------|
  | `resolve(node: ASTNode) -> None` | 空（仅 pass） | 2h — 入口：遍历 AST 收集泛型声明 |
  | `_resolve_type_parameters(node: TypeParameterNode) -> List[TypeParameter]` | 空 | 1h — 解析类型参数列表 |
  | `_resolve_constraints(node: TypeConstraintNode) -> List[TypeConstraint]` | 空 | 1h — 解析 where 约束子句 |
  | `instantiate_generic(node: GenericInstanceNode, type_args: List[str]) -> FunctionInstance` | 返回 None | 2h — 调用 GenericFunction.instantiate() |
  | `_check_constraints(type_param: TypeParameter, type_arg: str) -> bool` | 空 | 1.5h — 约束求解 |

- **具体工作**:
  1. 实现 `resolve()` 作为主入口，遍历 ProgramNode 收集所有 `GenericFunctionDefNode` 和 `GenericClassDefNode`
  2. 对每个泛型声明调用 `GenericManager.register_generic_function/type()`
  3. 处理 `GenericInstanceNode`（如 `最大值<整数型>(a, b)`），执行类型推导 + 实例化
  4. 约束检查委托给已有的 `TypeConstraint._check_constraint_satisfied()`
- **产出**: GenericResolver 可将 AST 中的泛型声明注册到 GenericManager，并将泛型调用点替换为特化版本
- **预估工时**: 7.5h
- **依赖**: 无

#### 任务 G.02 — Monomorphizer 单态化引擎

- **优先级**: 🔴 P0
- **文件**: `src/zhc/semantic/generics.py`
- **现状**: `Monomorphizer` 类存在，所有方法为空
- **需实现的方法**:

  | 方法签名 | 当前状态 | 工作量 |
  |---------|---------|--------|
  | `monomorphize(program: ProgramNode) -> ProgramNode` | 空 | 3h — 主入口：扫描所有泛型调用点 |
  | `monomorphize_function(func: GenericFunctionDefNode, type_args: List[str]) -> FunctionDeclNode` | 空 | 3h — 为给定类型参数生成特化函数副本 |
  | `monomorphize_class(cls: GenericClassDefNode, type_args: List[str]) -> StructDeclNode` | 空 | 3h — 为给定类型参数生成特化类副本 |
  | `_generate_specialized_copy(node: ASTNode, substitutions: Dict[str, str]) -> ASTNode` | 空 | 2h — AST 深拷贝 + 类型参数替换 |

- **具体工作**:
  1. 遍历 AST 找出所有 `GenericInstanceNode`（泛型实例化调用点）
  2. 对每个调用点做类型推导（从上下文推断类型实参）
  3. 调用 `_generate_specialized_copy()` 克隆泛型函数/类的 AST
  4. 在克隆的 AST中将所有类型参数名替换为实际类型名
  5. 将特化后的函数/类插入回 AST（使用 mangled name，如 `最大值__整数型`）
  6. 将原始 `GenericInstanceNode` 替换为普通 `CallExprNode`（指向特化函数）
- **关键细节**:
  - 名称修饰规则已在 `GenericFunction.get_mangled_name()` 中定义：`{name}__{arg1}_{arg2}`
  - `GenericType._substitute_type()` 已有字符串替换实现可复用
  - `_clone_body()` 目前返回原 body（TODO），需要实现真正的深拷贝
- **预估工时**: 11h
- **依赖**: G.01（需要 GenericResolver 先完成类型注册）

#### 任务 G.03 — Semantic Analyzer 集成泛型流程

- **优先级**: 🔴 P0
- **文件**: `src/zhc/semantic/semantic_analyzer.py`
- **现状**: 有 `visit_generic_function_def()` 和 `visit_generic_class_def()` 但可能只是简单转发
- **需添加/修改的内容**:

  | 项目 | 说明 |
  |------|------|
  | 在 `_analyze_node()` 分发中确认 `GENERIC_FUNCTION_DEF` / `GENERIC_CLASS_DEF` / `GENERIC_INSTANCE` 已正确路由 | 确保泛型节点不被遗漏 |
  | `visit_generic_function_def()` | 收集类型参数和约束 → 注册到 GenericManager |
  | `visit_generic_class_def()` | 同上，注册泛型类型 |
  | `visit_generic_instance()` | 类型推导 → 触发 Monomorphizer.monomorphize_function() |
  | 在语义分析流程末尾调用 `Monomorphizer.monomorphize(program)` | 单态化传递（在 IR 生成之前） |

- **具体工作**:
  1. 在 `SemanticAnalyzer.__init__()` 中初始化 `self._generic_resolver = GenericResolver()` 和 `self._monomorphizer = Monomorphizer()`
  2. 确保 visit 方法正确提取 AST 节点信息并传递给 Resolver
  3. 在 analyze() 末尾（IR 生成前）执行全程序单态化
- **预估工时**: 4h
- **依赖**: G.01, G.02

### P1 — 重要功能完善

#### 任务 G.04 — 泛型 IR 操作码扩展

- **优先级**: 🟡 P1
- **文件**: `src/zhc/ir/opcodes.py`
- **现状**: 无任何泛型专用操作码
- **需新增的操作码**:

  ```python
  # ========== 泛型 ==========
  GENERIC_DEF = ("generic_def", "泛型", "泛型定义", False, False)        # 泛型函数/类定义标记
  GENERIC_INSTANTIATE = ("generic_instantiate", "泛型", "泛型实例化", False, True)  # 实例化泛型
  TYPE_PARAM_BIND = ("type_param_bind", "泛型", "类型参数绑定", False, False)     # 绑定类型参数
  SPECIALIZE = ("specialize", "泛型", "特化生成", False, True)           # 生成特化版本
  ```

- **说明**: 这些操作码用于 IR 中间表示记录泛型实例化过程。单态化后大部分泛型调用会变成普通 CALL，但在调试/优化时需要保留实例化痕迹。
- **预估工时**: 0.5h
- **依赖**: 无（可并行于 G.01/G.02）

#### 任务 G.06 — Backend 单态化支持

- **优先级**: 🟡 P1
- **文件**: 
  - `src/zhc/backend/generic_strategies.py` （新建）
  - `src/zhc/backend/c_backend.py` （修改）
  - `src/zhc/backend/llvm_backend.py` （修改）
- **现状**: 所有后端均无泛型/单态化相关代码
- **需创建的策略**:

  | 策略类名 | 功能 | 对应 Opcode |
  |---------|------|------------|
  | `GenericDefStrategy` | 输出泛型定义元数据注释 | GENERIC_DEF |
  | `GenericInstantiateStrategy` | 生成实例化代码或调用特化函数 | GENERIC_INSTANTIATE |
  | `SpecializeStrategy` | 生成特化版本的函数/类 | SPECIALIZE |
  | `TypeParamBindStrategy` | 绑定类型参数到具体类型 | TYPE_PARAM_BIND |

- **注册函数**: `register_generic_strategies(factory) -> None`
- **C 后端输出示例**:
  ```c
  /* 特化: 最大值__整数型 */
  整数型 最大值__整数型(整数型 a, 整数型 b) {
      return (a > b) ? a : b;
  }
  ```
- **LLVM 后端**: 特化函数使用 mangled name 作为 LLVM function name
- **预估工时**: 6h
- **依赖**: G.04, G.05
