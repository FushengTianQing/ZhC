#### 任务 M.02 — 模式匹配 IR 操作码

- **优先级**: 🔴 P0
- **文件**: `src/zhc/ir/opcodes.py`
- **现状**: 无 MATCH/SWITCH/CASE/PATTERN 相关操作码
  - 注：已有 `SWITCH = ("switch", ...)` 用于 C-style switch，但模式匹配需要更精细的操作码
- **需新增的操作码**:

  ```python
  # ========== 模式匹配 ==========
  MATCH = ("match", "模式匹配", "匹配开始", False, False)   # match 作用域开始
  CASE = ("case", "模式匹配", "模式分支", False, False)    # 一个 case 分支开始
  PATTERN_BIND = ("pattern_bind", "模式匹配", "模式绑定", False, True)   # 绑定模式变量
  PATTERN_TEST = ("pattern_test", "模式匹配", "模式测试", False, True)  # 测试值是否匹配模式
  PATTERN_GUARD = ("pattern_guard", "模式匹配", "守卫条件", False, True) # 守卫表达式求值
  ```

- **设计说明**:
  - `MATCH`: 标记 match 表达式作用域入口
  - `CASE`: 标记每个匹配分支
  - `PATTERN_BIND`: 将匹配成功的变量绑定到寄存器（如解构 `点{x,y}` 中的 x, y）
  - `PATTERN_TEST`: 执行运行时模式匹配测试（用于构造器/字面量/嵌套模式）
  - `PATTERN_GUARD`: 计算 when 守卫表达式的布尔值
- **预估工时**: 0.5h
- **依赖**: 无

#### 任务 M.03 — IR Generator visit_match_expr

- **优先级**: 🔴 P0
- **文件**: `src/zhc/ir/ir_generator.py`
- **现状**: **不存在** `visit_match_expr()` 方法。MatchExpr 到达 IR Generator 时会被静默忽略或报错。
- **需实现的完整方法链**:

  | 方法签名 | 功能 | 工作量 |
  |---------|------|--------|
  | `visit_match_expr(self, node: MatchExpr)` | **主入口**：生成 match 作用的 IR，包含 scruting 表达式求值 + 分支分发 | 3h |
  | `_emit_pattern_test(self, pattern: Pattern, scrutinee_reg: str) -> str` | 发出 PATTERN_TEST 指令序列，返回是否匹配的结果寄存器 | 2h |
  | `_emit_pattern_bind(self, pattern: Pattern, scrutinee_reg: str)` | 发出 PATTERN_BIND 指令序列，提取变量绑定 | 2h |
  | `_emit_guard(self, guard_expr: Node) -> str` | 发出 PATTERN_GUARD 指令，计算守卫条件 | 1h |

- **核心降级策略**（match → if/else 链 或 jump table）:
  1. **简单模式**（字面量/通配符）：降级为 `switch-case` 或 `if-else` 链
  2. **构造器模式**（Some(x)/None）：降级为 `tag check + if-else`（先检查 discriminant，再递归处理子模式）
  3. **解构模式**（点{x, y}）：降级为 `GEP + LOAD` 序列提取字段 + 递归匹配
  4. **OR 模式**（x \| 1 \| 2）：展开为多个并列测试
  5. **Guard 模式**：在模式匹配成功后追加额外的条件跳转

- **IR 输出示例**（伪 IR）:
  ```
  %scrutinee = call get_value()
  ; === Case 1: Some(x) ===
  %tag = pattern_test CONSTRUCTOR(Some), %scrutinee
  jz %tag, :case2
  %x = pattern_bind FIELD(0), %scrutinee
  ; ... use x ...
  jmp :match_end
  ; === Case 2: None ===
  :case2
  %tag2 = pattern_test CONSTRUCTOR(None), %scrutinee
  jz %tag2, :match_fail
  ; ... none body ...
  jmp :match_end
  ; === Fallthrough ===
  :match_fail
  call throw_match_error(%scrutinee)
  :match_end
  ```

- **import 补充**:
  ```python
  from zhc.parser.ast_nodes import MatchExpr, PatternCase
  from zhc.semantic.pattern_matching import (
      WildcardPattern, LiteralPattern, VariablePattern,
      ConstructorPattern, GuardPattern, NestedPattern, OrPattern
  )
  ```
- **预估工时**: 8h
- **依赖**: M.02

#### 任务 M.04 — Backend 模式匹配代码生成策略

- **优先级**: 🔴 P0
- **文件**:
  - `src/zhc/backend/pattern_strategies.py` （新建）
  - `src/zhc/backend/c_backend.py` （注册）
  - `src/zhc/backend/llvm_backend.py` （注册）
- **现状**: 所有后端无法生成模式匹配代码
- **需创建的策略**:

  | 策略类名 | 功能 | 对应 Opcode |
  |---------|------|------------|
  | `MatchStrategy` | 生成 match 作用域（C: `{ /* match */ }`；LLVM: basic block） | MATCH |
  | `CaseStrategy` | 生成 case 分支标签/头部 | CASE |
  | `PatternTestStrategy` | 生成模式测试代码 | PATTERN_TEST |
  | `PatternBindStrategy` | 生成变量绑定代码（GEP+LOAD / struct member access） | PATTERN_BIND |
  | `PatternGuardStrategy` | 生成守卫条件代码 | PATTERN_GUARD |

- **C 后端输出示例**:
  ```c
  /* match (value) {
   *   case Some(x): ...
   *   case None: ...
   * } */
  {
      zhc_any_t* _scrutinee = value;
      /* Case 1: Some(x) */
      if (_scrutinee->tag == TAG_SOME) {
          整数型 x = (整数型)_scrutinee->fields[0];
          /* body */
          goto _match_end;
      }
      /* Case 2: None */
      if (_scrutinee->tag == TAG_NONE) {
          /* body */
          goto _match_end;
      }
      /* 非穷尽 fallback */
      zhc_match_error(_scrutinee);
      _match_end: (void)0;
  }
  ```

- **LLVM 后端输出**: 使用 `switch` on discriminant + 基本块拆分
- **注册函数**: `register_pattern_strategies(factory) -> None`
- **预估工时**: 8h
- **依赖**: M.02, M.03
