## 3. 为什么是预留功能

### 3.1 代码本身是完整的

`GenericInstantiator` 实现了完整的实例化逻辑：

```python
class GenericInstantiator:
    def instantiate_type(self, generic_type, type_args, context):
        # 1. 检查参数数量
        # 2. 检查缓存
        # 3. 检查循环依赖
        # 4. 检查约束
        # 5. 创建类型映射
        # 6. 实例化成员
        # 7. 创建实例
        # 8. 缓存实例
```

### 3.2 核心问题：没有调用点

实例化需要触发条件，但编译流程中没有调用 `GenericInstantiator` 的地方：

```
当前编译流程（简化）:
Parser → 解析泛型声明 → SemanticAnalyzer → 类型检查 → IR/后端

缺少的环节:
SemanticAnalyzer._infer_expr_type(CallExprNode{最大值(a, b)})
    ↓
    应该检测到最大值是泛型函数
    ↓
    从实参类型推导 T = 整数型
    ↓
    调用 instantiator.instantiate_function(generic_func, ["整数型"])
    ↓
    注册实例化后的函数
```

### 3.3 集成的技术障碍

| 障碍 | 说明 | 严重程度 |
|------|------|----------|
| **调用点缺失** | `SemanticAnalyzer` 没有泛型实例化的调用逻辑 | 🔴 高 |
| **类型推导依赖** | 需要从实参类型推导 `T`，依赖 `TypeInferenceEngine` | 🟡 中 |
| **AST 深拷贝** | `_instantiate_body()` 标注 `TODO`，需克隆函数体 | 🟡 中 |
| **代码生成集成** | 实例化后产生新函数，需在 IR/后端处理 | 🟡 中 |

```python
def _infer_expr_type(self, node: ASTNode):
    if node.node_type == ASTNodeType.CALL_EXPR:
        func_name = node.name
        
        # 检查是否是泛型函数
        if self.generic_manager.is_generic_function(func_name):
            # 从实参推导类型参数
            arg_types = [self._infer_expr_type(arg) for arg in node.args]
            type_args = self._infer_generic_args(func_name, arg_types)
            
            # 实例化
            instance = self.instantiator.instantiate_function(
                self.generic_manager.get_generic_functions(func_name)[0],
                type_args
            )
            
            # 注册实例化函数
            self._register_function_instance(instance)
            
            return instance.specialized_return_type
    
    # ... 原有逻辑
```

**Step 4：完成 AST 深拷贝**

```python
# generic_instantiator.py
def _instantiate_body(self, body: ASTNode, type_mapping: Dict[str, str], context) -> ASTNode:
    """实例化函数体"""
    # 深拷贝 AST
    import copy
    new_body = copy.deepcopy(body)
    
    # 遍历并替换类型引用
    self._replace_type_references(new_body, type_mapping)
    
    return new_body
```
