# 代码转换器设计文档
## 中文C编译器模块系统转换模块

### 📋 概述

代码转换器是中文C编译器第三阶段的核心组件，负责将中文模块语法转换为标准的C代码。本文档详细描述转换器的设计架构、转换规则和实现细节。

### 🔧 转换规则

#### 1. 模块声明转换

**输入语法**: `模块 模块名 { ... }`

**转换规则**:
1. 生成头文件保护宏: `#ifndef __模块名_H__`
2. 添加C++兼容声明: `extern "C" { ... }`
3. 根据可见性区域转换内容

**示例转换**:
```c
// 输入
模块 数学库 {
    公开:
        函数 加法(...) { ... }
}

// 输出头文件
#ifndef __数学库_H__
#define __数学库_H__

#ifdef __cplusplus
extern "C" {
#endif

int 数学库_加法(...);

#ifdef __cplusplus
}
#endif

#endif /* __数学库_H__ */
```

#### 2. 导入语句转换

**输入语法**: `导入 模块名;`

**转换规则**: `导入 模块名;` → `#include "模块名.h"`

**处理逻辑**:
1. 检查模块是否已定义
2. 生成标准#include指令
3. 记录模块依赖关系

#### 3. 符号转换规则

##### 公开符号
- **规则**: `模块名_符号名`
- **示例**: `数学库.加法` → `数学库_加法`

##### 私有符号
- **规则**: `static 模块名_符号名`
- **示例**: `数学库.内部变量` → `static 数学库_内部变量`

##### 保护符号
- **规则**: 暂与私有相同（待扩展）
- **示例**: `数学库.受保护函数` → `static 数学库_受保护函数`

#### 4. 类型关键字转换

**类型映射表**:
| 中文类型 | C类型 | 说明 |
|:---|:---|:---|
| 整数型 | int | 32位整数 |
| 字符型 | char | 8位字符 |
| 浮点型 | float | 单精度浮点数 |
| 双精度浮点型 | double | 双精度浮点数 |
| 逻辑型 | _Bool | C99布尔类型 |
| 长整数型 | long | 长整数 |
| 短整数型 | short | 短整数 |
| 无类型 | void | 无返回值 |
| 结构体 | struct | 结构体声明 |
| 共用体 | union | 联合体声明 |
| 枚举 | enum | 枚举类型 |

**中文前缀支持**:
- `中文整数型` → `int`
- `中文浮点型` → `float`

#### 5. 函数定义转换

**输入语法**: `函数 函数名(参数列表) -> 返回类型 { ... }`

**转换规则**:
1. 转换参数类型
2. 转换返回类型
3. 添加模块前缀
4. 处理可见性修饰符

**示例**:
```c
// 输入
函数 最大值(整数型 a, 整数型 b) -> 整数型

// 输出
int 工具库_最大值(int a, int b)
```

#### 6. 变量定义转换

**输入语法**:
- `类型 变量名;`
- `类型 变量名 = 初始值;`

**转换规则**:
1. 转换类型关键字
2. 添加模块前缀
3. 处理可见性（私有变量添加static）
4. 保留初始值表达式

**示例**:
```c
// 输入
整数型 计数器 = 0;

// 输出（公开）
int 数学库_计数器 = 0;

// 输出（私有）
static int 数学库_内部计数器 = 0;
```

### 🔄 转换流程

#### 完整转换步骤
1. **预处理**: 读取文件，分割行，处理编码
2. **解析阶段**: 使用ModuleParser分析语法结构
3. **符号收集**: 收集所有符号及其可见性信息
4. **转换阶段**: 应用转换规则生成C代码
5. **验证阶段**: 检查转换结果的有效性
6. **输出阶段**: 写入头文件和源文件

#### 增量转换支持
支持基于文件修改时间的增量转换：
```python
def should_convert(input_file: str, output_file: str) -> bool:
    """检查是否需要重新转换"""
    if not os.path.exists(output_file):
        return True
        
    input_mtime = os.path.getmtime(input_file)
    output_mtime = os.path.getmtime(output_file)
    
    return input_mtime > output_mtime
```

### 📊 性能考虑

#### 时间复杂度分析
| 操作 | 时间复杂度 | 说明 |
|:---|:---|:---|
| 语法解析 | O(n) | 线性扫描输入文件 |
| 符号查找 | O(1) | 使用哈希表存储符号 |
| 类型转换 | O(1) | 字典查找类型映射 |
| 文件写入 | O(m) | 线性写入输出文件 |

#### 内存使用优化
1. **流式处理**: 逐行处理，避免加载整个文件到内存
2. **符号去重**: 使用集合存储符号，避免重复
3. **缓存机制**: 缓存常用转换结果
4. **内存池**: 重用对象减少分配开销

### 🎯 设计原则

1. **正确性优先**: 确保转换结果的语法正确性
2. **可读性**: 生成的C代码应易于阅读和理解
3. **可维护性**: 代码结构清晰，易于扩展和维护
4. **性能**: 在保证正确性的前提下优化性能
5. **兼容性**: 支持多种C标准（C89/C99/C11）
