# 中文C编译器 - 面向对象编程指南

## 概述

本指南介绍如何使用中文C编译器的类系统进行面向对象编程。

## 类定义

### 基本语法

```
类 类名 {
    公开:
    属性:
        类型 属性名;

    方法:
        函数 方法名() -> 返回类型 { }
}
```

### 示例

```
类 学生 {
    公开:
    属性:
        字符串型 姓名;
        整数型 年龄;
        浮点型 成绩;

    方法:
        函数 构造函数(字符串型 名, 整数型 龄) -> 空型 {
            姓名 = 名;
            年龄 = 龄;
        }

        函数 获取信息() -> 字符串型 {
            返回 姓名;
        }

        函数 设置成绩(浮点型 分) -> 空型 {
            成绩 = 分;
        }
}
```

## 可见性控制

| 关键字 | 说明 |
|:---|:---|
| 公开: | 公有成员，可被外部访问 |
| 私有: | 私有成员，只能类内部访问 |
| 保护: | 保护成员，子类可访问 |

### 示例

```
类 用户 {
    公开:
    属性:
        字符串型 用户名;

    私有:
    属性:
        字符串型 密码;

    保护:
    属性:
        整数型 权限等级;
}
```

## 继承

### 基本语法

```
类 子类名 : 父类名 {
    属性:
        类型 新属性;
}
```

### 示例

```
类 形状 {
    公开:
    属性:
        整数型 x;
        整数型 y;

    方法:
        函数 绘制() -> 空型 { }
}

类 圆形 : 形状 {
    属性:
        浮点型 半径;

    方法:
        函数 获取面积() -> 浮点型 {
            返回 3.14159 * 半径 * 半径;
        }
}
```

## 多态与虚函数

### 虚函数表

编译器自动为包含虚函数的类生成虚函数表：

```
类 形状 {
    方法:
        虚函数 绘制() -> 空型;
        虚函数 获取面积() -> 浮点型;
}
```

### 动态分派

通过虚函数表实现运行时多态：

```c
// 编译器生成
typedef struct 形状_vtable {
    void (**methods)(void *self);
} 形状_vtable_t;

typedef struct 形状 {
    形状_vtable_t *vptr;
    int x;
    int y;
} 形状_t;
```

## 运算符重载

### 语法

```
操作符 操作符名(参数类型) -> 返回类型
```

### 示例

```
类 向量 {
    属性:
        浮点型 x;
        浮点型 y;

    方法:
        操作符 加(向量 v) -> 向量 {
            向量 结果;
            结果.x = x + v.x;
            结果.y = y + v.y;
            返回 结果;
        }

        操作符 乘(浮点型 s) -> 向量 {
            向量 结果;
            结果.x = x * s;
            结果.y = y * s;
            返回 结果;
        }
}
```

## 构造函数与析构函数

### 构造函数

```
函数 构造函数(参数) -> 空型 { }
```

### 析构函数

```
函数 析构函数() -> 空型 { }
```

### 示例

```
类 动态数组 {
    属性:
        整数型* 数据;
        整数型 长度;

    方法:
        函数 构造函数(整数型 大小) -> 空型 {
            数据 = 分配(大小 * sizeof(整数型));
            长度 = 大小;
        }

        函数 析构函数() -> 空型 {
            释放(数据);
        }
}
```

## 完整示例

```
类 学生 {
    公开:
    属性:
        字符串型 姓名;
        整数型 年龄;

    私有:
    属性:
        浮点型 成绩;

    方法:
        函数 构造函数(字符串型 名, 整数型 龄) -> 空型 {
            姓名 = 名;
            年龄 = 龄;
            成绩 = 0.0;
        }

        函数 获取信息() -> 字符串型 {
            返回 姓名;
        }

        函数 设置成绩(浮点型 分) -> 空型 {
            成绩 = 分;
        }
}

类 大学生 : 学生 {
    属性:
        字符串型 专业;

    方法:
        函数 构造函数(字符串型 名, 整数型 龄, 字符串型 专) -> 空型 {
            姓名 = 名;
            年龄 = 龄;
            专业 = 专;
        }
}
```

## 最佳实践

1. **封装**: 将数据成员设为私有，通过方法访问
2. **继承层次**: 保持继承层次不过深（建议不超过3层）
3. **虚函数**: 仅在需要多态时使用虚函数，避免不必要的开销
4. **运算符重载**: 仅在语义清晰时使用
